# TeamClaw App

A full-stack starter: **TanStack Start** (Node SSR via Nitro) + **Postgres**.

Generated by TeamClaw. Edit freely — your AI sessions can build on top of this.

## Structure
- `src/routes/__root.tsx` — document shell (HTML, head, scripts)
- `src/routes/index.tsx` — demo route: a loader that `SELECT`s `items` and a
  server fn that `INSERT`s one (real DB read + write)
- `src/db.ts` — pooled Postgres client (`postgres`) reading `DATABASE_URL`, plus
  `ensureSchema()` (idempotent, runs the schema on first boot)
- `src/router.tsx` — router factory
- `db/schema.sql` — **source of truth** for the schema. The DDL is mirrored
  (inlined) in `src/db.ts` so it is bundled and runnable at runtime; keep the two
  in sync.
- `vite.config.ts` — selects the Nitro **`node-server`** preset

## Build & run (production / FC)
```bash
pnpm build                       # emits .output/ (Nitro node-server preset)
PORT=9000 DATABASE_URL=postgres://... node .output/server/index.mjs
```

## FC deploy shape (Alibaba FC custom runtime, `custom.debian10`)
The deployed artifact is the `.output/` tree. The daemon zips `.output` and FC
runs it as a long-lived HTTP server:

- **Startup command**: `node .output/server/index.mjs`
- **Port**: listens on `process.env.PORT` (FC injects `PORT=9000`), binds
  `0.0.0.0` (Nitro node-server default). FC's `customRuntimeConfig.port` must
  match (`9000`).
- **DB**: `DATABASE_URL` is injected as a server-only env var (never
  `VITE_`-prefixed). Its role has a pinned `search_path` to this app's own
  schema, so unqualified table names resolve there.
- **Schema on boot**: the app runs `ensureSchema()` on first request (the
  provisioner does NOT run app migrations). The DDL is idempotent
  (`create table if not exists`).

Build notes:
- `vite.config.ts` sets `build.target: 'esnext'` so esbuild does not
  down-transpile the Node server bundle to the browser default (es2020), which
  otherwise fails on modern destructuring in the Nitro entry.
- Externalized deps (incl. `postgres`) land in `.output/server/node_modules`;
  the daemon zips the whole `.output` tree, so the artifact is self-contained.
- The generated `src/routeTree.gen.ts` is gitignored (created by the Start vite
  plugin on `dev`/`build`).
